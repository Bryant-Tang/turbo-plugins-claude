# Test Plan

> 所有 `test-n.md` 的 `n` 都要替換成實際驗證任務編號，例如 `test-1.md`、`test-2.md`。
> 此測試計畫涵蓋 [goal.md](./goal.md) 中所有目標的最終總驗證，存放於 spec 資料夾根目錄。

## 目的

- 對 `goal.md` 中所有目標進行整體最終驗證。
- 將最後總驗證拆成可在單一 chat session 內完成的小型驗證任務。
- 將驗證前置、證據規則、任務順序集中管理，避免驗證要求散落在 `goal.md` 或 `goal-<id>/plan.md`。

## 證據規則

- Browser 任務必須以實際系統畫面截圖作為主要證據，並直接在 `test-n.md` 使用 Markdown 圖片語法嵌入，且 `n` 必須替換成實際驗證任務編號。
- 截圖只能是系統畫面，不得使用人工加註、後製、合成、拼接或替代畫面。
- 若任務目標是驗證 UI 顯示，API、資料庫查詢、DOM 擷取或 console 結果只能作為輔助證據，不能取代系統畫面。
- 非 Browser 任務必須提供檔案與行號連結作為主要證據。
- 每個驗證任務都必須回填實際結果與證據，不能只寫預期結果。

## 任務拆分原則

- 每個驗證任務只處理一個主要目標。
- 每個驗證任務都要能在單一 session 內完成。
- 每個 `test-n.md` 都要寫清楚目標、前置、操作步驟、預期結果、實際結果、證據與阻塞條件，且 `n` 必須替換成實際驗證任務編號。
